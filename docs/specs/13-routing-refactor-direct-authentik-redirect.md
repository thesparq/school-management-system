# Unit 13: Routing Refactor – Direct Authentik Redirect

## Goal

Eliminate the intermediate `/login` page by redirecting unauthenticated users directly to Authentik's OIDC authorization URL (server-side 302). Move the dashboard from `/dashboard` to `/`, remove the static landing page at `/`, and flatten the route hierarchy by deleting the `(auth)` route group.

## Design

- **No unauthenticated page routes remain.** The landing page at `/` and the `/login` page are deleted. The only page routes are the dashboard (`/`) and admin pages (`/admin/...`), both protected.
- **Auth guard lives in root `+layout.server.ts`.** When `event.locals.user` is null, it calls `generateAuthUrl()` to produce the Authentik authorization URL, sets `oauth_state` and `oauth_code_verifier` cookies (same PKCE flow as before), then issues a 302 redirect directly to Authentik. No client-side fetch to `/api/auth/login` is needed — the `/api/auth/login` endpoint is deleted.
- **The `(auth)` route group is removed entirely.** The sidebar/navbar layout moves from `(auth)/+layout.svelte` to `routes/+layout.svelte`. Since unauthenticated users are redirected before the root layout renders, the layout doesn't need conditional logic — it always renders the sidebar shell.
- **Admin routes move up one level.** `routes/(auth)/admin/...` → `routes/admin/...`. The admin role guard in `admin/users/+layout.server.ts` is unchanged (defense-in-depth).
- **Callback and logout redirect targets change.** `/api/auth/callback` redirects to `/` instead of `/dashboard`. The `post_logout_redirect_uri` in `getEndSessionUrl` points to `/` instead of `/login`.
- **No visual changes to the UI.** The dashboard looks identical; the sidebar, breadcrumbs, avatar dropdown, and logout button are unchanged. Only the URL structure and redirect flow change.

### 3. Create Root Layout Server Load

**New file:** `frontend/src/routes/+layout.server.ts`

```typescript
import { redirect } from '@sveltejs/kit';
import type { LayoutServerLoad } from './$types';
import { generateAuthUrl } from '$lib/server/authentik';

const SECURE = process.env.NODE_ENV === 'production';

export const load: LayoutServerLoad = async (event) => {
	if (!event.locals.user) {
		const { url, state, codeVerifier } = await generateAuthUrl();

		event.cookies.set('oauth_state', state, {
			httpOnly: true,
			sameSite: 'lax',
			path: '/api/auth/callback',
			maxAge: 300,
			secure: SECURE
		});

		event.cookies.set('oauth_code_verifier', codeVerifier, {
			httpOnly: true,
			sameSite: 'lax',
			path: '/api/auth/callback',
			maxAge: 300,
			secure: SECURE
		});

		redirect(302, url);
	}

	return { user: event.locals.user };
};
```

### 5. Move Dashboard to Root

**New file:** `frontend/src/routes/+page.server.ts`

Content from the existing `(auth)/dashboard/+page.server.ts`:

```typescript
import { proxyToGateway } from '$lib/server/golem';
import type { PageServerLoad } from './$types';

export const load: PageServerLoad = async (event) => {
	const user = event.locals.user;
	if (!user) return { initialized: true };

	if (user.roles.includes('admin')) return { initialized: true };

	const result = await proxyToGateway('/gateway/check-initialization', user.id);
	if (result.error?.code === 'NOT_INITIALIZED') {
		return { initialized: false };
	}
	return { initialized: true };
};
```
