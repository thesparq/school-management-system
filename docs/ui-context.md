# UI Context ‚Äì Design Tokens

## Color Tokens

All colors are defined as CSS custom properties via Tailwind CSS v4‚Äôs `@theme` directive. The palette uses a calm, trustworthy blue as the primary brand color and a warm amber as the secondary accent. Success and error colors follow standard educational conventions (green for correct/completed, red for incorrect/overdue). Surface neutrals provide consistent backgrounds and text.

## Typography

| Token | Font Stack | Usage |
|:---|:---|:---|
| `--font-sans` | `'Inter', system-ui, -apple-system, Segoe UI, Roboto, sans-serif` | Body text, UI labels, buttons, forms, breadcrumbs, navigation |
| `--font-mono` | `'JetBrains Mono', 'Fira Code', monospace` | Code snippets, technical content, data tables (numeric alignment) |
| `--font-display` | `'Inter', system-ui, sans-serif` (with tighter letter-spacing) | Page headings, lesson titles, card headers |

All fonts should be loaded from a CDN or self-hosted; fallbacks are standard system fonts for performance.

## Border Radius Scale

| Token | Value | Usage |
|:---|:---|:---|
| `--radius-sm` | `0.25rem` (4px) | Small elements: checkboxes, tags, badges, tiny buttons |
| `--radius-md` | `0.375rem` (6px) | Default: buttons, inputs, cards, dropdowns |
| `--radius-lg` | `0.5rem` (8px) | Larger cards, modals, dialogs |
| `--radius-xl` | `0.75rem` (12px) | Sidebar, wide containers |
| `--radius-full` | `9999px` | Pills, avatars, circular indicators |

Use these consistently across all shadcn-svelte components; they map directly to the library‚Äôs `radius` props.

## AI & Accent Color Variants

No special AI accent color is required for the MVP. The primary blue is sufficient for AI-related UI elements (e.g., an AI sparkle icon can remain blue). If a distinct ‚ÄúAI‚Äù identity is needed later, introduce a **purple/violet** accent (`#8b5cf6` ‚Äì `#7c3aed`) reserved only for AI-generated content labels, badges, and subtle glows; it must never replace the semantic colors for success or error.

## Implementation Note (Tailwind v4)

In `app.css`, define the tokens like this:

```css
@import "tailwindcss";

@theme {
  --color-primary-50: #eff6ff;
  --color-primary-100: #dbeafe;
  /* ‚Ä¶ all other tokens ‚Ä¶ */
  --font-sans: 'Inter', system-ui, sans-serif;
  --radius-md: 0.375rem;
}
```

Then use them as `bg-primary-500`, `text-surface-800`, `rounded-md`, etc. The design system stays consistent and fully under version control.
